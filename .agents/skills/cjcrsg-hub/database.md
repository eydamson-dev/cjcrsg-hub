# Database Schema Reference

## Overview

Convex uses a document-relational database. We define schemas in TypeScript with validators for type safety.

## Core Tables

### 1. attendees

Stores church members and visitors.

```typescript
{
  _id: string                    // Auto-generated by Convex
  firstName: string              // Required
  lastName: string               // Required
  email?: string                 // Optional
  phone?: string                 // Optional
  dateOfBirth?: number           // Optional (timestamp)
  address?: string               // Optional
  status: 'member' | 'visitor' | 'inactive'  // Required
  joinDate?: number              // Optional (timestamp)
  notes?: string                 // Optional
  createdAt: number              // Auto-set
  updatedAt: number              // Auto-set
}
```

**Indexes:**

```typescript
.index('by_status', ['status'])
.index('by_email', ['email'])
.index('by_phone', ['phone'])
.searchIndex('search_name', {
  searchField: 'fullName',
  filterFields: ['status']
})
```

**Usage:**

- List all members: `by_status` index
- Search by name: `search_name` index
- Lookup by email/phone: `by_email`, `by_phone`

### 3. events

Scheduled events (services, retreats, etc.).

```typescript
{
  _id: string
  name: string                   // Event name
  eventTypeId: string            // Reference to event_types
  description?: string           // Optional details
  date: number                   // Event date (timestamp, midnight)
  startTime?: string             // "09:00" (24-hour format)
  endTime?: string               // "11:00" (24-hour format)
  location?: string              // e.g., "Main Sanctuary"
  isActive: boolean              // Cancelled events = false
  createdAt: number
}
```

**Indexes:**

```typescript
.index('by_date', ['date'])
.index('by_event_type', ['eventTypeId'])
.index('by_active_date', ['isActive', 'date'])
```

**Usage:**

- List upcoming: `by_active_date` with `isActive: true`, order by date
- Filter by type: `by_event_type`

### 4. attendance_records

Who attended which event.

```typescript
{
  _id: string
  eventId: string                 // Reference to events
  attendeeId: string              // Reference to attendees
  checkedInAt: number            // Timestamp when marked present
  checkedInBy: string            // Admin user ID who recorded
  notes?: string                 // Optional notes
}
```

**Indexes:**

```typescript
.index('by_event', ['eventId', 'checkedInAt'])
.index('by_attendee', ['attendeeId', 'checkedInAt'])
.index('by_event_attendee', ['eventId', 'attendeeId'])
```

**Usage:**

- Get attendance for event: `by_event`
- Get attendance history for person: `by_attendee`
- Check if already attended: `by_event_attendee` (unique)

## Query Patterns

### 1. List with Pagination

```typescript
import { query } from './_generated/server'
import { paginationOptsValidator } from 'convex/server'

export const list = query({
  args: {
    paginationOpts: paginationOptsValidator,
  },
  handler: async (ctx, args) => {
    return await ctx.db
      .query('attendees')
      .order('desc') // Newest first
      .paginate(args.paginationOpts)
  },
})
```

### 2. Search with Filters

```typescript
export const search = query({
  args: {
    query: v.string(),
    status: v.optional(v.string()),
  },
  handler: async (ctx, args) => {
    let results = ctx.db
      .query('attendees')
      .withSearchIndex('search_name', (q) => q.search('fullName', args.query))

    if (args.status) {
      results = results.filter((q) => q.eq(q.field('status'), args.status))
    }

    return await results.take(50)
  },
})
```

### 3. Get Related Data

```typescript
export const getEventWithAttendance = query({
  args: { eventId: v.id('events') },
  handler: async (ctx, args) => {
    const event = await ctx.db.get(args.eventId)
    if (!event) return null

    const attendance = await ctx.db
      .query('attendance_records')
      .withIndex('by_event', (q) => q.eq('eventId', args.eventId))
      .collect()

    // Get attendee details
    const attendees = await Promise.all(
      attendance.map((record) => ctx.db.get(record.attendeeId)),
    )

    return { event, attendees: attendees.filter(Boolean) }
  },
})
```

### 4. Time-based Queries

```typescript
// Upcoming events
export const listUpcoming = query({
  args: {},
  handler: async (ctx) => {
    const now = Date.now()
    return await ctx.db
      .query('events')
      .withIndex('by_active_date', (q) =>
        q.eq('isActive', true).gte('date', now),
      )
      .order('asc')
      .take(20)
  },
})
```

## Mutation Patterns

### Create

```typescript
import { mutation } from './_generated/server'

export const create = mutation({
  args: {
    firstName: v.string(),
    lastName: v.string(),
    email: v.optional(v.string()),
    phone: v.optional(v.string()),
    status: v.union(
      v.literal('member'),
      v.literal('visitor'),
      v.literal('inactive'),
    ),
  },
  handler: async (ctx, args) => {
    const now = Date.now()
    const id = await ctx.db.insert('attendees', {
      ...args,
      createdAt: now,
      updatedAt: now,
    })
    return id
  },
})
```

### Update

```typescript
export const update = mutation({
  args: {
    id: v.id('attendees'),
    firstName: v.optional(v.string()),
    lastName: v.optional(v.string()),
    email: v.optional(v.string()),
    // ... other optional fields
  },
  handler: async (ctx, args) => {
    const { id, ...updates } = args
    await ctx.db.patch(id, {
      ...updates,
      updatedAt: Date.now(),
    })
  },
})
```

### Delete (Soft)

```typescript
export const archive = mutation({
  args: { id: v.id('attendees') },
  handler: async (ctx, args) => {
    await ctx.db.patch(args.id, {
      status: 'inactive',
      updatedAt: Date.now(),
    })
  },
})
```

### Attendance Check-in

```typescript
export const checkIn = mutation({
  args: {
    eventId: v.id('events'),
    attendeeId: v.id('attendees'),
  },
  handler: async (ctx, args) => {
    // Get current user (admin)
    const userId = await ctx.auth.getUserId()
    if (!userId) throw new Error('Not authenticated')

    // Check if already checked in
    const existing = await ctx.db
      .query('attendance_records')
      .withIndex('by_event_attendee', (q) =>
        q.eq('eventId', args.eventId).eq('attendeeId', args.attendeeId),
      )
      .first()

    if (existing) {
      throw new Error('Already checked in')
    }

    // Create attendance record
    await ctx.db.insert('attendance_records', {
      ...args,
      checkedInAt: Date.now(),
      checkedInBy: userId,
    })
  },
})
```

## Data Types

### Convex Validators

```typescript
import { v } from 'convex/values'

// String
v.string()

// Number (timestamp, count, etc.)
v.number()

// Boolean
v.boolean()

// Optional
v.optional(v.string())

// Union (enums)
v.union(v.literal('member'), v.literal('visitor'), v.literal('inactive'))

// ID reference
v.id('attendees')
v.id('events')

// Objects
v.object({
  firstName: v.string(),
  lastName: v.string(),
})

// Arrays
v.array(v.string())

// Any (avoid if possible)
v.any()
```

## Best Practices

1. **Always use indexes** for filtered queries
2. **Use searchIndex** for text search (not filter)
3. **Paginate** large lists (don't use `.collect()` on big tables)
4. **Validate inputs** with Convex validators
5. **Set timestamps** (createdAt, updatedAt) on all mutations
6. **Soft delete** with `isActive` flag instead of hard delete
7. **Check auth** in mutations that modify data
8. **Use transactions** when modifying multiple documents
